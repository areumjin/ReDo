# ReDo — Design System

> 디자이너·크리에이터를 위한 레퍼런스 관리 앱의 시각 언어
> Linear의 정밀함 + ReDo의 창의적 감성

---

## 철학

**"도구는 사라지고, 레퍼런스만 남는다"**

ReDo의 UI는 콘텐츠를 방해하지 않는다. Linear에서 빌려온 울트라 미니멀리즘을 기반으로, 불필요한 장식을 제거하고 사용자의 시선이 항상 *저장한 레퍼런스*로 향하게 설계한다.

### 3가지 핵심 원칙

| 원칙 | 설명 |
|------|------|
| **Precision** | 1px도 허투루 쓰지 않는다. 여백은 의도적이고, 정렬은 수학적이다 |
| **Restraint** | 브랜드 컬러(`#6A70FF`)는 딱 필요한 곳에만. 나머지는 무채색 |
| **Clarity** | 한국어 UI에서도 Linear급의 읽기 편한 정보 밀도를 유지한다 |

---

## 컬러 시스템

### 브랜드

| 역할 | 값 | 사용처 |
|------|----|--------|
| Brand | `#6A70FF` | 활성 탭, 선택 상태, CTA 버튼, 진행 바 |
| Brand Dark | `#8B8FFF` | 다크모드 브랜드 |
| Brand Tint | `#EEEFFE` | 선택된 행 배경, hover 상태 |

> **Linear 원칙 적용:** 브랜드 컬러는 *active/selected* 상태에만 사용한다. 일반 텍스트, 배경, 테두리에 절대 사용하지 않는다.

### 상태 색상 (Impact Colors)

작은 영역에만 강하게 사용한다. 뱃지, 아이콘, 상태 점(dot) 수준.

| 상태 | Light | Dark | 사용처 |
|------|-------|------|--------|
| Success | `#1D9E75` | `#3DC49A` | 저장 완료, 동기화 성공 |
| Warning | `#EF9F27` | `#F5B84A` | 만료 임박, 주의 필요 |
| Danger | `#E24B4A` | `#FF7573` | 삭제, 오류, 접근 불가 |
| Info | `#3B82F6` | `#64B5F6` | 알림, 도움말 |

### 텍스트 계층 (5단계 무채색)

순수 검정(`#000`)은 절대 사용하지 않는다.

| 단계 | Light | Dark | 역할 |
|------|-------|------|------|
| Strong | `#1A1A1A` | `#F0F0F0` | 최고 강조 (거의 미사용) |
| Primary | `#3C3C3C` | `#E0E0E0` | 제목, 핵심 수치 |
| Secondary | `#6A6A6A` | `#A0A0A0` | 레이블, 캡션 |
| Tertiary | `#7A7A7A` | `#808080` | 날짜, 부연 설명 |
| Disabled | `#9B9B9B` | `#555555` | 비활성, placeholder |

### 배경 시스템

| 레이어 | Light | Dark | 역할 |
|--------|-------|------|------|
| Page | `#FFFFFF` | `#121212` | 앱 전체 배경 |
| Card | `#FFFFFF` | `#1E1E1E` | 카드 내부 |
| Subtle | `#FAFAFA` | `#252525` | 리스트 행, hover |
| Muted | `#EBEBF0` | `rgba(255,255,255,0.08)` | 구분선, 진행 바 트랙 |

> **Linear 원칙 적용:** 다크모드에서 카드는 페이지보다 *밝게*. 그림자 대신 미묘한 명도 차이로 깊이를 표현한다.

### Context Box (저장 이유 박스)

ReDo만의 고유 UI 요소 — 레퍼런스를 저장한 *이유*를 기록하는 공간.

| 속성 | Light | Dark |
|------|-------|------|
| 배경 | `#EEEFFE` | `rgba(106,112,255,0.12)` |
| 텍스트 | `#3C3489` | `#A5A8FF` |
| 토큰 | `bg-context-box-bg` / `text-context-box-text` | — |

```tsx
<div className="rounded-xl px-4 py-3 bg-context-box-bg">
  <p className="text-[13px] text-context-box-text leading-relaxed">
    {reason}
  </p>
</div>
```

## 타이포그래피

**서체:** Pretendard (한국어 UI 최적화) → Inter → system-ui

Linear처럼 타이포그래피 자체가 디자인 요소다. 크기 차이와 굵기 차이만으로 계층을 만든다.

### 크기 스케일

| 용도 | 크기 | 굵기 | 사용처 |
|------|------|------|--------|
| 페이지 타이틀 | `24px` | 700 | 최상단 h1 |
| 섹션 제목 | `18px` | 700 | 카드 헤더 |
| 리스트 제목 | `14px` | 700 | 아이템 이름 |
| 본문 | `14px` | 400 | 설명 텍스트 |
| 캡션 | `13px` | 400 | 날짜, 출처, 태그 |
| 배지 | `12px` | 700 | 카테고리, 상태 레이블 |
| 마이크로 | `11px` | 500 | 상태 텍스트, 단위 |

> **금지:** CSS 변수로 폰트 크기 정의 (`--text-sm` 등). Tailwind v4의 `--text-*` 네임스페이스와 충돌한다. 항상 `text-[14px]` 형태의 명시적 px 값 사용.

### 행간 규칙

| 크기 | 행간 | 이유 |
|------|------|------|
| 24px+ | `leading-none` (1.0) | 제목은 타이트하게 |
| 18–20px | `leading-snug` (1.35) | 섹션 제목 |
| 13–17px | `leading-normal` (1.5) | 본문 가독성 |
| 11–12px | `leading-relaxed` (1.65) | 작은 텍스트는 숨 공간 필요 |

## 여백 & 레이아웃

### 모바일 기준

```
최대 너비: 430px (iPhone Pro Max 기준)
페이지 좌우 여백: px-6 (24px)
섹션 간격: space-y-6 (24px)
하단 내비 높이: 64px + safe-area
```

### 카드 vs 그리드 여백 규칙

| 상황 | 클래스 | 효과 |
|------|--------|------|
| 단독 카드 (SectionCard, HeroCard) | `mx-6` | 카드가 양쪽 마진으로 떠 있음 |
| 2열 그리드, 캐러셀 | `px-6` | 컨텐츠가 엣지까지 꽉 참 |

### 간격 단위

6px 배수 시스템만 사용.

```
6px  → gap-1.5, p-1.5
12px → gap-3, p-3
18px → gap-4.5
24px → gap-6, p-6
```

## 컴포넌트 원칙

### 카드

모든 콘텐츠는 카드 안에. 배경 위에 직접 놓지 않는다.

```
radius: rounded-2xl (16px)
shadow (light): 0 1px 3px rgba(0,0,0,0.04)
shadow (dark): none — 명도 차이로 대체
border (dark): rgba(255,255,255,0.08)
```

### 버튼

| 종류 | 용도 |
|------|------|
| Primary | 저장, 추가 등 핵심 액션 1개/화면 |
| Secondary | 보조 액션, 취소 |
| Ghost | 툴바, 내비게이션 내 액션 |
| Destructive | 삭제, 제거 |

- 높이: 최소 44px (터치 타겟 규칙)
- 브랜드 버튼 배경: `#6A70FF` — 한 화면에 1개 원칙

### 입력 필드

Linear처럼 테두리를 최소화한다.

```
기본 배경: #F3F3F5 (--input-background)
포커스: 테두리 color → brand (#6A70FF)
에러: 테두리 color → danger (#E24B4A)
radius: rounded-md (8px)
```

### 태그 / 배지

```
배경: brand-tint (#EEEFFE)
텍스트: brand (#6A70FF)
크기: text-[12px] font-bold
radius: rounded-full
padding: px-2 py-0.5
```

## 그림자 시스템

≤ 8% 불투명도. 보이면 너무 강한 것이다.

| 토큰 | 값 | 사용처 |
|------|----|--------|
| `--shadow-card` | `0 1px 3px rgba(0,0,0,0.04)` | 기본 카드 |
| `--shadow-card-hover` | `0 2px 4px rgba(0,0,0,0.08)` | hover 상태 |
| `--shadow-elevated` | `0 4px 12px rgba(0,0,0,0.08)` | 떠 있는 패널 |
| `--shadow-modal` | `0 8px 24px rgba(0,0,0,0.12)` | 모달, 바텀시트 |

## 다크모드

`.dark` 클래스 전략. 시스템 설정이 아닌 앱 내 토글로 제어.

```css
@custom-variant dark (&:is(.dark *));
```

**Linear 다크모드 원칙 3가지:**
1. 카드는 페이지 배경보다 밝다 (`#1E1E1E` > `#121212`)
2. 그림자 없음 — 대신 `rgba(255,255,255,0.08)` 테두리
3. 브랜드 컬러는 밝게 (`#8B8FFF`) — 어두운 배경에서 채도 유지

## 모션 & 애니메이션

빠르고 절제된다. Linear처럼 *느린 애니메이션은 없다*.

| 토큰 | 값 | 사용처 |
|------|----|--------|
| `--duration-fast` | 100ms | hover, 색상 전환 |
| `--duration-normal` | 200ms | 패널 열기/닫기 |
| `--duration-slow` | 350ms | 페이지 전환 |
| `--ease-spring` | `cubic-bezier(0.34, 1.56, 0.64, 1)` | 체크, 토글 |

`prefers-reduced-motion` 설정 시 모든 애니메이션 자동 비활성화.

## 접근성

| 규칙 | 기준 |
|------|------|
| 터치 타겟 | 최소 44×44px |
| 본문 텍스트 대비 | 7:1 이상 (WCAG AA) |
| 보조 텍스트 대비 | 4.5:1 이상 |
| 포커스 링 | `focus-visible:ring-2 ring-brand` |
| 색상만으로 상태 표현 금지 | 아이콘 또는 텍스트와 함께 |

## 금지 사항 (Linear 기준에서 가져온 것들)

```
✗ 순수 검정 #000 사용
✗ 한 화면에 브랜드 컬러 2개 이상 버튼
✗ 카드 없이 배경 위에 직접 콘텐츠
✗ 8% 초과 그림자 불투명도
✗ CSS 변수로 폰트 크기 정의 (--text-* 충돌)
✗ px-4, mx-4, px-8 — px-6/mx-6만 허용
✗ 44px 미만 터치 타겟
✗ 의미적 토큰 대신 하드코딩 hex (컴포넌트 내부에서)
```
